
フレームワークについて

・JSF
　：Webアプリケーションのインターフェイスを構築するためのフレームワーク。
　　アプリケーションのビジネスロジックとプレゼンテーションデザインの分離が可能
　　テキストフィールドやボタン、セレクトボックス等をJSPカスタムタグライブラリを使用して
　　表現することが可能。さらにそれらのコンポーネントに対してステート情報の保持や
　　入力値のチェック、型の変換、イベントの制御、データモデルへのマッピング
　　といった機能の付与が可能。
　：必要なもの
	Java SE（Javaのコンパイラーと実行環境）
	NetBeans IDE（ソフト開発ツール、Webサーバー・サーブレットコンテナ

・Spring Framework
　：変更に強い（依存性の注入）。1つ1つのプログラムを独立させやすくなる＝
　　変更する場合、最低限の変更で済むので開発が楽になる。
　　テストが簡単。「Spring MVC Test」という専用のテストプログラムを使うことができます。
　　これは、1つ1つの処理がうまくいっているかをテスト（単体テスト）するのではなく、
　　無事に一連の処理を終えるかどうかをテスト（結合テスト）することができる。
　：必要なもの
	Spring Tool Suite(STS)
	※Javaのバージョン(64bit?32bit?)に注意

・Apache Struts
　：オープンソース。MVCモデルに基づいている。
　　HTMLタグの中に<%と%>で囲まれたスクリプトレットであるJavaソースコードを混在させる必要なく読みやすく
　　洗練されたコーディングをできるようになる。
　　サーブレットから設定ファイル（struts-config.xml）に指定されたアクションクラスを
　　呼び出し、アクションクラスの実行結果により、画面表示を決める事ができ
　：必要なもの
	Struts
　　※虚弱性の問題があったことがある
